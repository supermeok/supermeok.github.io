Agent是什么

The Redefine Team Lv1

最近在 AI 领域，频率出现agent这个概念，有人说它是 “下一代 AI 形态”，那但到底什么是 Agent，和我们平时用的大模型有什么联系? 本文带你理清 Agent 的核心逻辑，并介绍开发Agent常用的工具。

AI Agent基础概念

定义

在第一次接触一个新事物时，想要理解并掌握它,首先应知道它的定义，《人工智能》一书中给出了Agent的宽泛定义：

任何通过传感器（sensor）感知环境（environment）并通过执行器(actuator)作用于该环境的事物都可以被视作智能体

比如：

  • 自动驾驶：通过传感器感知道路状况，自主决策加速、转弯或刹车
  • AI助教：捕捉学生的发音，表达，提出意见

Agent早在1997年就已提出，历经多轮发展和演变，它的关键就是能感知环境并做出响应，而如今爆火的 Agent，核心是 “基于大模型” 的智能体 ， 这也是它能成为 “下一代 AI 形态” 的关键。

基于LLM的agent

首先要明确，大模型并不等同于Agent，我们平时使用的Deepseek,豆包，ChatGpt等大模型，交互多停留在 提问并回复的聊天页面。它们能理解需求、给出答案，但无法主动感知环境、执行跨平台操作 —— 比如你让大模型写邮件并发送，它做不到，但 Agent 可以。

举个实际场景：
如果你想做一份生日派对策划

  • 用传统大模型：需手动分步引导，先问生日派对需要准备什么，再问 如何订蛋糕，接着确认邀请函设计模板，每一步都依赖人工触发。；
  • 用 Agent：你只需要说 :“帮我策划一场 10 人的生日派对，预算 2000 元，时间下周六”，它会自动完成所有步骤，最后给你一份完整的执行方案。

如今爆火的Agent,是基于LLM(Large Language Model)的智能体，智能体技术演变的最新阶段 。得益于LLM的快速发展，在规划、推理、知识获取等方面的强大能力，同时与人类表现出有效的自然语言交互等优势，所以现在的Agent便是以LLM作为大脑，承担控制器的角色

简单来说，Agent 是具备 “自主感知、决策、执行” 能力的 AI 系统，它不需要我们每一步都发指令，只要你给它一个明确目标，它就能自己规划步骤、调用资源、解决问题，甚至在过程中调整策略。

AI Agent核心技术

Agent通常由以下四个核心组件构成：

Agent = LLM + 记忆 (Memory)+ 任务规划(Planning) + 工具使用(Tools)

四大核心组件

  1. LLM：提供核心的语言理解、推理与生成能力，是整个Agent的“大脑”、控制器。
  2. 任务规划：对复杂任务借助大模型进行分解、规划和调度，如著名的Chain-of-thought, React等方式，及时观察子任务执行的结果与反馈，对任务及时调整。
  3. 工具使用：让大模型能感知环境，并作用于环境。根据决策结果执行具体的动作或指令,与外部工具进行交互，不在局限于聊天窗口，如主动查看日历，运行代码，网页搜索等
  4. 记忆：让大模型能长期学习，存储经验和知识。作为“存储器”，存储短期的记忆（如一次任务过程中的多次人类交互）或长期记忆（如记录使用者的任务历史、个人信息、兴趣便好等）

后面的文章会详细介绍每一个组件的详细信息

OpenAI SDK框架

OpenAI SDK 是开发Agent最常用的工具之一。它封装了 OpenAI 大模型的核心能力，简化了 “记忆管理、任务规划、工具调用” 的开发流程，让开发者无需关注底层逻辑，就能快速搭建 Agent 原型，且无需本地下载大模型，只需api_key就可以进行快速开发。
